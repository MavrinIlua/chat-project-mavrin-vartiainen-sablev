# ChatApp — Веб-чат в реальном времени

> Простой и удобный чат прямо в браузере — без установки приложений.

---

## Что это такое?

**ChatApp** — учебный веб-чат где несколько пользователей могут переписываться в реальном времени.
Достаточно открыть браузер, придумать никнейм и пароль — и ты уже в чате.

---

## Возможности

- **Умное приветствие** — "Доброе утро", "Добрый день", "Добрый вечер" или "Доброй ночи" в зависимости от времени
- **Вход по никнейму и паролю** — первый раз вводишь данные и регистрируешься автоматически
- **История сообщений** — переписка сохраняется в базе данных
- **Время и автор** — у каждого сообщения видно кто и когда написал
- **Группировка по дням** — дата показывается один раз при переходе на новый день
- **Защита от чужого входа** — чужой никнейм без пароля не займёшь
- **Хеширование паролей** — безопасное хранение паролей через werkzeug
- **Динамический чат** — обновление без перезагрузки страницы
- **Чат 1-на-1** — выбор собеседника и переписка с конкретным пользователем
- **MySQL-поддержка** — стабильная работа с удалённой базой данных

## Как работает база данных

### SQLite-версия (app.py)

- **Файл БД:** `chat.db` (автоматически создаётся в папке проекта)
- **Структура:**
  - `users`: id, nickname, password (хэшированный через `werkzeug`)
  - `messages`: id, nickname, text, timestamp (формат: "15.04.2026|14:30")
- **История:** Последние 100 сообщений, сортировка по времени
- **Как проверить:** Запустите сервер и откройте браузер → `http://127.0.0.1:5000`

### MySQL-версия (app_mysql.py)

- **Подключение:** Удалённый сервер MySQL (185.114.247.43)
- **База данных:** `sch688_maga1`
- **Структура:**
  - `users`: id, name, surname, login, password, created_at, last_seen, is_active
  - `messages`: id, owner_id, deliver_id, text, timestamp, is_read
- **Как проверить:** Запустите `python app_mysql.py`, откройте браузер → `http://127.0.0.1:5000`

## Как работает динамический чат

### Шаг 1: Определение текущего пользователя

```html
<body data-nickname="{{ nickname }}">
```

### Шаг 2: Загрузка сообщений

```javascript
function loadMessages() {
    fetch("/messages")  // → JSON список сообщений
        .then(response => response.json())
        .then(messages => {
            // Очистка и рендер
        });
}
```

### Шаг 3: Отправка сообщения

```javascript
fetch("/send", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body: "message=" + encodeURIComponent(text)
}).then(() => {
    input.value = "";
    loadMessages();  // Обновление
});
```

### Шаг 4: Автоматическое обновление

```javascript
setInterval(loadMessages, 3000);  // Каждые 3 секунды
```

## Решение типичных проблем

### Ошибка "ModuleNotFoundError: No module named 'flask'"

```bash
.venv\Scripts\python.exe -m pip install flask
```

### Ошибка подключения к MySQL БД

Убедитесь, что создан файл `.env` с правильными параметрами:

```
DB_HOST=185.114.247.43
DB_NAME=sch688_maga1
DB_USER=sch688_maga1
DB_PASS=Fqx8irSU
DB_PORT=3306
```

### Ошибка "SyntaxError: invalid syntax" в Python

Проверьте версию Python:
```
python --version
```
Должно быть **3.10 или новее**.

### CSS не подгружается

1. Очистите кэш браузера (Ctrl+Shift+R)
2. Проверьте, что файл `static/style.css` существует
3. Убедитесь, что сервер запущен

### Страница не обновляется при отправке сообщения

1. Проверьте консоль браузера (F12 → Console)
2. Убедитесь, что в `chat.js` есть вызов `loadMessages()` после отправки
3. Проверьте, что маршрут `/send` работает (вводите в браузере: `http://127.0.0.1:5000/send`)
