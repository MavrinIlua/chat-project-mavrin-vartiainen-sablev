## Архитектура проекта

```
chat-project/
│

│   └── chat.html         ← страница чата с сообщениями (Илья + Евгений + Анна)

    └── chat.js           ← динамика без перезагрузки (Анна)
```

## Как устроено приложение — схема работы

```
Браузер пользователя
        │
        │  HTTP-запрос (открыть страницу, отправить форму)
        ▼
   app.py (Flask)
        │
        ├── маршрут "/"       → показывает index.html
        ├── маршрут "/login"  → проверяет пользователя в БД
        ├── маршрут "/chat"   → показывает chat.html + сообщения из БД
        ├── маршрут "/send"   → сохраняет сообщение в БД
        └── маршрут "/logout" → очищает сессию
        │
        ▼
   chat.db (SQLite)
        │
        ├── таблица users    (id, nickname, password)
        └── таблица messages (id, nickname, text, timestamp)
```

## Как запустить проект

1. Открыть папку проекта в VS Code
2. Открыть терминал (Terminal → New Terminal)
3. Убедиться что активировано виртуальное окружение — в терминале должно быть `(venv)`
4. Запустить командой:
   ```
   python app.py
   ```
5. Открыть браузер и перейти по адресу: `http://127.0.0.1:5000`
6. Остановить сервер: `Ctrl+C` в терминале

## Задачи Евгения

**Бэклог ID 5 — История в БД**

Структура таблицы `messages` уже создана в `app.py` (функция `init_db`).
Маршрут `/send` уже сохраняет сообщения. Маршрут `/chat` уже их читает.

Что нужно сделать:
- Проверить что история отображается корректно при входе
- При желании — добавить ограничение на количество сообщений в выдаче,
  например последние 100: изменить запрос в `/chat` на:
  ```python
  messages = conn.execute(
      "SELECT * FROM messages ORDER BY id DESC LIMIT 100"
  ).fetchall()
  messages = list(reversed(messages))  # перевернуть чтобы новые были внизу
  ```

**Бэклог ID 7 — Отображение времени и автора**

Время уже записывается в БД при отправке (в маршруте `/send`).
Автор и время уже выводятся в `chat.html`.

Что нужно сделать:
- Убедиться что формат времени удобный (сейчас: `15.01.2024 14:30`)
- Если нужно изменить формат — поменять строку в `app.py` в маршруте `/send`:
  ```python
  # Текущий формат:
  timestamp = datetime.datetime.now().strftime("%d.%m.%Y %H:%M")

  # Альтернативы:
  # "%H:%M"              →  14:30  (только время)
  # "%d.%m %H:%M"        →  15.01 14:30
  # "%d.%m.%Y %H:%M:%S"  →  15.01.2024 14:30:05  (с секундами)
  ```

## Задачи Анны

**Бэклог ID 6 — Динамическая отправка (файл `static/chat.js`)**

Сейчас при отправке сообщения страница перезагружается целиком.
Задача — сделать как в Telegram: сообщение появляется без перезагрузки.

Шаги:
1. Перехватить отправку формы через JavaScript
2. Отправить данные на сервер тихо (через `fetch`)
3. Добавить новое сообщение в блок `#messages` без перезагрузки

Пример структуры кода для `chat.js`:
```javascript
// Находим форму на странице
const form = document.getElementById("message-form");

// Перехватываем отправку
form.addEventListener("submit", function(e) {
    e.preventDefault(); // отменяем стандартную перезагрузку

    // Берём текст из поля ввода
    const input = document.getElementById("message-input");
    const text = input.value.trim();
    if (!text) return;

    // Отправляем на сервер в фоне
    fetch("/send", {
        method: "POST",
        headers: { "Content-Type": "application/x-www-form-urlencoded" },
        body: "message=" + encodeURIComponent(text)
    }).then(() => {
        input.value = ""; // очищаем поле
        loadMessages(); // обновляем список сообщений
    });
});
```

Для `loadMessages()` нужно будет добавить маршрут в `app.py` который
возвращает сообщения в формате JSON — обсудить с Ильёй.

**Оформление (`static/style.css`)**

Что стилизовать (классы уже прописаны в `chat.html`):
```
body           — фон страницы, шрифт
h1             — заголовок
#messages      — блок сообщений: высота, прокрутка (overflow-y: scroll)
.message       — одно сообщение: рамка, отступы, скруглённые углы
.author        — никнейм автора: жирный, цвет
.time          — время: маленький, серый
.text          — текст сообщения
form           — форма внизу: фиксированное положение или просто внизу
input[type=text] — поле ввода: широкое, красивое
button         — кнопка отправить
a              — ссылка "Выйти"
```

## Частые ошибки и решения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| "Не удаётся подключиться к сайту" | Сервер не запущен | Запустить `python app.py` |
| TemplateSyntaxError | Пропущен `{% endif %}` или `{% endfor %}` | Проверить все открытые теги в HTML |
| "Неверный пароль" при первом входе | Случайная опечатка | Это нормально — просто введи снова |
| `chat.db` выглядит как мусор | Это бинарный файл | Не открывать в редакторе, всё нормально |
